

1. gitflow
	1) main브랜치
	=>실제 제품으로 나갈 수 있는 상태
	
	2) develop 브랜치
		=> main 브랜치를 기반으로 만들어진 브랜치
		=> 다음 배포에 나갈 내용을 메인에서 머지할 수 없으므로 진행중인 개발 내용이 merge될 브랜치
		=> 다음 출시 version을 개발해서 합치는 브랜치
	
	3) feature 브랜치
		=> 각각의 개발자들이 자기의 개발을 하기 위해서 만드는 작업브랜치
		=> feature 브랜치에서 작업이 끝나면 push하고 merge가 끝나면 삭제될 브랜치
		=> develop 브랜치를 기반으로 만들고 develop 브랜치에 merge
		=> 기능을 개발하는 브랜치
		
	4) realease 브랜치
		=> develop 브랜치를 기반으로 만들기
		=> 이번 출시 버전을 준비하는 브랜치
		=> 작업이 완료되고 배포시점이 왔을 때 생성
		=> Q&A나 여러 작업을 마무리하고 main 브랜치와 merge
		
	5) hotfix 브랜치
		=> 배포직전에 혹은 운영중에 긴급하게 발생할 수 있는 오류나 이슈가 있는 경우 정리하기 위한 브랜치
		=> 출시직전에 버그를 수정하기 위한 브랜치
		=> release를 기준으로 만드는 것이 일반적
		=> main 브랜치와 merge
